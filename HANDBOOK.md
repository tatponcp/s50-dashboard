คู่มือฉบับอ่านเข้าใจได้โดยไม่ต้องเก่งโค้ด — อธิบายว่าระบบทำงานยังไง
ดูแลยังไง และพังแล้วซ่อมยังไง

## 1. โปรเจกต์นี้คืออะไร

เว็บ dashboard สรุปภาพตลาด **SET50 Index Options** แบบ End-of-Day
ให้เทรดเดอร์เปิดดูตอนเช้าบนมือถือ/คอมได้ทันที โดย**ข้อมูลอัปเดตเอง
ทุกเย็นวันทำการ ไม่ต้องมีใครทำอะไรเลย**

```
ทุกเย็นวันทำการ 19:30 น. (อัตโนมัติ)
┌─────────────┐    ดึงข้อมูล    ┌──────────────┐    commit     ┌──────────────┐
│  TFEX API   │ ─────────────▶ │ GitHub Actions│ ────────────▶ │  data.json   │
│ (ตลาดอนุพันธ์)│               │  รัน scraper  │               │  ใน GitHub   │
└─────────────┘                └──────────────┘               └──────┬───────┘
                                                                     │ เผยแพร่เอง
                                                                     ▼
        ลูกค้าเปิดดูได้ตลอดเวลา ◀──────────────  https://tatponcp.github.io/s50-dashboard/
```

**ลิงก์ที่ต้องรู้:**

| อะไร | ที่ไหน |
|---|---|
| 🌐 เว็บสำหรับลูกค้า | https://tatponcp.github.io/s50-dashboard/ |
| 📦 โค้ดทั้งหมด (repo) | https://github.com/tatponcp/s50-dashboard |
| ⚙️ ห้องเครื่อง (ดู/สั่งรันการอัปเดต) | https://github.com/tatponcp/s50-dashboard/actions |

**หน้าเว็บแบ่งเป็น 3 แท็บ** (ดีไซน์ใหม่ ก.ค. 2026 — ธีมน้ำเงิน/ทอง ฟอนต์ IBM Plex):

- **Overview** — ราคาปิด SET50 + Expected Range, กราฟ OI รายสไตรค์,
  การ์ดตัวชี้วัด 4 ใบ (Max Pain, PCR, ATM IV, DTE)
  และตารางรายสไตรค์ (มีแถบสีในเซลล์บอกสัดส่วน OI + ป้าย ATM / MAX PAIN)
- **OI Change** — กราฟ OI เพิ่ม/ลด เทียบ 1D / 5D / 10D แยกฝั่ง Call/Put
  (ช่วงแรกจะขึ้นข้อความ "ยังไม่มีข้อมูลสแนปช็อต..." จนกว่าจะสะสม history — ปกติ)
- **IV Smile** — เส้นโครงสร้าง IV รายสไตรค์เทียบสองซีรีส์ (เส้นประ = ซีรีส์ไกล)

## 2. ไฟล์ไหนทำอะไร

```
s50-dashboard/
├── index.html            ← หน้าเว็บ dashboard (โครงหน้า)
├── styles.css            ← หน้าตา/สี/ฟอนต์ทั้งหมด (แก้ธีมที่นี่ที่เดียว)
├── data.json             ← ข้อมูลตลาดวันล่าสุด + ประวัติย้อนหลัง (ห้ามแก้มือ!)
├── js/
│   ├── metrics.js        ← สูตรคำนวณ (Max Pain, PCR, ...)
│   └── dashboard.js      ← วาดกราฟ/ตาราง/animation
├── scraper/
│   ├── tfex_scraper.py   ← ตัวดึงข้อมูลจาก TFEX (หัวใจของระบบ)
│   └── requirements.txt  ← รายชื่อ library ที่ scraper ใช้
├── .github/workflows/
│   └── update-data.yml   ← ตารางเวลาอัตโนมัติ "ทุกเย็นให้รัน scraper แล้ว commit"
├── tools/
│   └── sheet-api.gs      ← (ทางเลือกสำรอง) กรอกข้อมูลผ่าน Google Sheet แทน
├── HANDBOOK.md           ← ไฟล์นี้
└── README.md             ← หน้าปกของ repo
```

หลักการจำง่ายๆ:
- **อยากแก้หน้าตา** → `styles.css` (สี/ระยะ/ฟอนต์รวมอยู่หัวไฟล์ ในส่วน `:root`)
- **อยากแก้ข้อมูล** → ไม่ต้องแก้ — ระบบดึงเองทุกเย็น
- **อยากแก้เวลา/วิธีดึงข้อมูล** → `scraper/tfex_scraper.py` และ `update-data.yml`

## 3. ระบบทำงานยังไงในแต่ละวัน

1. **19:30 น. (จ.-ศ.)** GitHub Actions ตื่นขึ้นมาเองตามตารางเวลา
2. รัน `scraper/tfex_scraper.py` ซึ่งจะ:
   - เรียก API ของ TFEX → ได้ข้อมูลออปชันทุกซีรีส์ ทุกสไตรค์
     (OI, Volume, IV, Delta, วันหมดอายุ) + ราคาปิด SET50
   - **เลือกซีรีส์อัตโนมัติ**: เอาเฉพาะซีรีส์รายไตรมาส (มี.ค./มิ.ย./ก.ย./ธ.ค.)
     ที่ยังไม่หมดอายุ 2 ตัวที่ใกล้สุด — ตอนนี้คือ U26, Z26 และพอ U26
     หมดอายุ ระบบจะสลับเป็น Z26, H27 **เองโดยไม่ต้องแก้โค้ด**
   - **ตรวจสอบข้อมูลก่อนบันทึก** (validation): จำนวนสไตรค์พอไหม
     OI รวมเป็นบวกไหม ราคา SET50 สมเหตุสมผลไหม —
     **ถ้าผิดปกติจะไม่เขียนทับของเดิม** เว็บจึงไม่มีวันแสดงข้อมูลขยะ
   - ย้ายข้อมูลเมื่อวานเข้า "history" (เก็บย้อนหลังสูงสุด 60 วัน)
3. commit `data.json` เวอร์ชันใหม่เข้า GitHub
4. GitHub Pages เผยแพร่ให้เอง ภายใน ~1-2 นาที ลูกค้ารีเฟรชก็เห็นข้อมูลใหม่

> 💡 ถ้าวันไหนเป็นวันหยุดตลาด TFEX จะส่งข้อมูลวันทำการล่าสุดมาให้
> ระบบฉลาดพอที่จะไม่บันทึกวันซ้ำ — history จะไม่มีวันหยุดปน

## 4. เช็คสุขภาพระบบ (ใช้เวลา 1 นาที)

เปิด https://github.com/tatponcp/s50-dashboard/actions แล้วดู:

- ✅ **เครื่องหมายถูกเขียว** ที่รายการล่าสุด = ระบบปกติ
- ❌ **กากบาทแดง** = รอบล่าสุดพัง → คลิกเข้าไปอ่าน log (ดูข้อ 5)
- ที่หน้าเว็บลูกค้า แถบหัวเว็บด้านขวามีคำว่า **"ข้อมูล ณ วันที่..."** —
  ถ้าเป็นวันทำการล่าสุดแปลว่าทุกอย่างเดินปกติ

## 5. ปัญหาที่พบบ่อย + วิธีแก้

| อาการ | สาเหตุที่เป็นไปได้ | วิธีแก้ |
|---|---|---|
| เว็บขึ้นข้อมูลวันเก่า | รอบอัตโนมัติเมื่อวานพัง | เข้า Actions → คลิก **"Update SET50 options data"** → ปุ่ม **"Run workflow"** → รอ 1-2 นาที |
| Actions ขึ้นกากบาทแดง มี "403" ใน log | TFEX บล็อก IP ของเครื่อง GitHub ชั่วคราว | กด Run workflow ซ้ำ ถ้ายังไม่หายให้ใช้ "แผนสำรอง" ด้านล่าง |
| Actions ขึ้น "Validation ไม่ผ่าน" | TFEX เปลี่ยนรูปแบบข้อมูล | ต้องแก้ scraper — ส่ง log ให้คนเขียนโค้ด/AI ช่วยดู (ระบบตั้งใจหยุดเองเพื่อไม่ให้ข้อมูลเสียขึ้นเว็บ) |
| แท็บ "OI Change" ขึ้น "ยังไม่มีข้อมูลสแนปช็อต..." | เหมือนข้างบน — ยังไม่มีข้อมูลเมื่อวานให้เทียบ | รอ 1 วันทำการ (กรอบ 5D/10D รอ 5/10 วัน) |
| เปิดไฟล์ index.html ในเครื่องแล้วเว็บว่าง | เบราว์เซอร์บล็อกการอ่าน data.json จาก `file://` | ดูวิธี "รันในเครื่อง" ข้อ 6 |

**แผนสำรองเมื่อ GitHub โดน TFEX บล็อกนานๆ** — รันจากเครื่องตัวเองแล้วส่งขึ้นไป:

```
(เปิด Terminal ในโฟลเดอร์โปรเจกต์)
python scraper/tfex_scraper.py
git add data.json
git commit -m "update data"
git push
```

หรือสลับไปใช้เส้นทาง Google Sheet: ทีมกรอกตัวเลขใน Sheet ทุกเย็น
แล้ว dashboard ดึงจาก Sheet โดยตรง — วิธีติดตั้งอยู่หัวไฟล์
`tools/sheet-api.gs` (ติดตั้งครั้งเดียว แล้ววาง URL ใน `js/dashboard.js`)

## 6. วิธีรันดูในเครื่องตัวเอง (ก่อนแก้อะไรจริง)

```
(เปิด Terminal ในโฟลเดอร์โปรเจกต์)
python -m http.server 8000
```

แล้วเปิดเบราว์เซอร์ไปที่ `http://localhost:8000` —
เหตุที่ต้องทำแบบนี้: เบราว์เซอร์ไม่ยอมให้หน้าเว็บที่เปิดจากไฟล์ตรงๆ
(`file://`) อ่านไฟล์ข้อมูล ต้องเสิร์ฟผ่าน server จำลองก่อน

อยากทดสอบ scraper เอง:

```
pip install requests        (ครั้งแรกครั้งเดียว)
python scraper/tfex_scraper.py
```

จะเห็นสรุปข้อมูลที่ดึงได้ในจอ ถ้าผ่าน validation มันจะเขียน `data.json` ให้

## 7. งานแก้ไขที่พบบ่อย (สูตรสำเร็จ)

| อยากทำอะไร | แก้ตรงไหน |
|---|---|
| เปลี่ยนสี Call/Put/พื้นหลัง | `styles.css` ส่วน `:root` บนสุด — สีทุกจุดรวมไว้ที่เดียว กราฟเปลี่ยนตามอัตโนมัติ |
| เปลี่ยนฟอนต์ | ลิงก์ Google Fonts ใน `index.html` + ตัวแปร `--font-sans` / `--font-mono` ใน `styles.css` |
| เปลี่ยนเวลาอัปเดต | `.github/workflows/update-data.yml` บรรทัด `cron:` (ระวัง: เขียนเป็นเวลา UTC = เวลาไทยลบ 7 ชม.) |
| เก็บซีรีส์เพิ่ม (เช่น 3 ตัว) | `scraper/tfex_scraper.py` ค่า `NUM_SERIES` |
| เก็บ history ยาวขึ้น | `scraper/tfex_scraper.py` ค่า `HISTORY_DAYS` |
| แก้ข้อความบนหน้าเว็บ | `index.html` (ข้อความส่วนใหญ่อยู่ในนี้ตรงๆ ค้นหาแล้วแก้ได้เลย) |

หลังแก้ → commit + push → Pages อัปเดตเว็บให้เองใน 1-2 นาที

## 8. ข้อควรรู้ / ข้อจำกัด

- **Repo เป็น public** (ข้อบังคับของ GitHub Pages ฟรี) — โค้ดและข้อมูล
  เปิดเผย ซึ่งโอเคเพราะเป็นข้อมูลตลาดสาธารณะ ไม่มีรหัสผ่าน/ความลับใดๆ
  ในโปรเจกต์ **ห้าม commit ความลับใดๆ ลง repo นี้เด็ดขาด**
- ข้อมูลเป็น **End-of-Day เท่านั้น** ไม่ใช่ realtime
- ตัวชี้วัดที่ต้องใช้ประวัติ (OI Change, ⚡ ผิดปกติ) จะค่อยๆ
  "ติดไฟ" เมื่อระบบสะสมข้อมูลจริงครบ: OI Change 1D ใช้ได้หลัง 1 วัน,
  ⚡ หลัง ~10 วัน
- IV Rank / Percentile และ Skew ถูกถอดออกจากหน้าเว็บชั่วคราว (ก.ค. 2026)
  เพราะยังใช้จริงไม่ได้ — สูตรยังอยู่ใน `js/metrics.js` ถ้าจะนำกลับมา
- ประวัติทุกวันถูกเก็บใน git — ถ้า data.json พังยังกู้คืนย้อนหลังได้เสมอ
  (เข้า GitHub → history ของไฟล์ → เลือกวันเก่า)
- dashboard **ไม่มีและห้ามมีคำแนะนำการลงทุน** — เป็นเครื่องมืออ่าน
  ตำแหน่งของตลาดเท่านั้น (มีประกาศท้ายหน้าเว็บแล้ว)

## 9. ศัพท์ในหน้าเว็บ (ฉบับย่อ)

| ตัวชี้วัด | อ่านว่าอะไร |
|---|---|
| **Max Pain** | ราคาที่ถ้าดัชนีปิดตรงนั้นวันหมดอายุ ผู้ถือออปชันทั้งตลาดเจ็บสุด — ราคามักถูก "ดึง" เข้าหาจุดนี้เมื่อใกล้หมดอายุ |
| **PCR** | Put OI ÷ Call OI — สูง = ตลาดกันขาลงเยอะ, ต่ำ = เก็งขาขึ้นเยอะ |
| **ATM IV** | ความผันผวนแฝงที่สไตรค์ใกล้ราคาปัจจุบัน — "ค่าความกลัว/ความคาดหวัง" ของตลาด |
| **Expected Range** | ช่วงราคาที่ตลาดให้โอกาส ~68% ว่าดัชนีจะปิดในนั้น ณ วันหมดอายุ |
| **Vol/OI > 1** | วอลุ่มวันนี้มากกว่าสถานะคงค้าง = มีเงินใหม่เข้าผิดปกติที่สไตรค์นั้น |
| **⚡ (ในกราฟ/ตาราง)** | OI สไตรค์นี้เปลี่ยนแรงผิดปกติเทียบกับพฤติกรรมของตัวมันเอง |
| **DTE** | จำนวนวันเหลือก่อนซีรีส์หมดอายุ — ต่ำกว่า 5 วันจะมีแบนเนอร์เตือนให้เริ่มดูซีรีส์ถัดไป |
